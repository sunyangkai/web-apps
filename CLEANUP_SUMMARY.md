## 🎯 新的配置方式

### 之前（已废弃）

**配置分散在多个 .env 文件中：**

```
.env.development          # 开发环境
.env.test                 # 测试环境
.env.production           # 生产环境
.env.local.user           # User 组开发
.env.local.sale           # Sale 组开发
.env.local.base           # Base 组开发
```

**问题：**
- ❌ 配置重复，域名散落在多个文件
- ❌ 修改测试环境域名需要改多个文件
- ❌ 添加新模块需要修改所有 .env 文件

---

### 现在（推荐）

**集中配置文件：** `config/remotes.config.js`

```javascript
// 环境域名映射（统一管理）
const DOMAIN_MAP = {
  local: 'http://localhost',
  dev: 'http://dev.example.com',
  test: 'http://test.example.com',
  staging: 'http://staging.example.com',
  production: 'https://cdn.example.com',
};

// 模块端口配置
const MODULE_PORTS = {
  base: 3000,
  user: 3001,
  sale: 3002,
};
```

**启动脚本：** `scripts/start.js`

根据参数自动生成配置：

```bash
node scripts/start.js --env=test --local=user
```

**优势：**
- ✅ 配置集中，易于管理
- ✅ 修改域名只需改一处
- ✅ 添加新模块只需加一行
- ✅ 自动生成 .env.temp，无需手动编辑

## 🚀 启动命令对比

### 之前

```bash
# 需要维护多个 .env 文件
pnpm dev                 # 使用 .env.development
pnpm dev:local-user      # 使用 .env.local.user
NODE_ENV=test pnpm build # 使用 .env.test
```

### 现在

```bash
# 通过脚本参数控制
pnpm dev                 # 全本地（自动生成配置）
pnpm dev:local-user      # User 组（自动生成配置）
pnpm dev:test            # 测试环境（自动生成配置）
```

**启动时自动生成 `.env.temp`：**

```env
# Auto-generated by scripts/start.js
NODE_ENV=development
PUBLIC_PATH=http://localhost:3100/
REMOTES={"base":"base@http://localhost:3000/remoteEntry.js","user":"user@http://localhost:3001/remoteEntry.js","sale":"sale@http://test.example.com:3002/remoteEntry.js"}
```

## 💡 后续维护

### 添加新环境

只需修改 `config/remotes.config.js`：

```javascript
const DOMAIN_MAP = {
  local: 'http://localhost',
  test: 'http://test.example.com',
  uat: 'http://uat.example.com',  // 新增
  production: 'https://cdn.example.com',
};
```

### 添加新模块

只需修改 `config/remotes.config.js`：

```javascript
const MODULE_PORTS = {
  base: 3000,
  user: 3001,
  sale: 3002,
  product: 3003,  // 新增
};
```

### 修改域名

只需修改 `config/remotes.config.js`：

```javascript
const DOMAIN_MAP = {
  test: 'http://new-test-domain.com',  // 修改
};
```

所有使用该环境的命令自动生效！
